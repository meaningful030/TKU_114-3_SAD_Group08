# 需求與畫面對照表（Requirements-to-Screen Mapping）

## 1. 對照範圍與狀態

- 對象：`revised_or_second_working_slice/` 的第二個可操作切片。
- 本次可操作需求：`FR-01～FR-05、FR-08`；`NFR-04` 基本可存取性；`NFR-01、NFR-05` 有部分／技術支援。
- 高優先但未實作：`FR-07` 與 `NFR-02` 排下一切片，對應規劃中的 `SCR-05`，不得在本表誤標為現有畫面。
- 後續需求：`FR-06` 排位衝突對應規劃中的 `SCR-04`，本次沒有入口。
- `SCR-03A、SCR-03B` 是同一主辦方畫面 `SCR-03` 的兩個可操作區；使用子編號可保留既有追溯矩陣中的 `SCR-03`，同時滿足每個操作區都有編號。
