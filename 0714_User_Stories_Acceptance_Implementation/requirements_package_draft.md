## 1. 專案名稱、問題與範圍摘要

### 1.1 專案名稱

同人誌販售會攤位管理系統「攤集」。

### 1.2 問題摘要

目前攤位申請、審核、排位與現場查攤分散在表單、email 和試算表中，容易發生漏件、審核結果難以回查、同一格位重複配置，以及配置表與公開地圖資訊不同步。系統希望提供一條可追蹤的流程，讓攤主確認送件及結果、主辦方集中審核，並讓參加者依明確資料查找攤位。

### 1.3 專案範圍

| 範圍類別 | 內容 |
|---|---|
| 本期需求範圍 | 攤位申請與驗證、申請編號及狀態、主辦方通過／退回、退回後修改原申請重送、手動配置衝突檢查、靜態地圖搜尋 |
| 本次已實作 | FR-01～FR-05、FR-08；NFR-04 基本語意；NFR-01 與 NFR-05 的既有支援 |
| 下一切片 | FR-07 靜態地圖搜尋及 NFR-02 資料一致性 |
| 後續切片 | FR-06 重複格位衝突；其他候選 NFR 依確認結果排入 |
| 本次／本期不做 | 正式登入、後端、資料庫、通知、金流、候補、多場次、票務、商品管理、R18 審查、工作排班、自動排位、互動式地圖 |
| 原型限制 | 原生 HTML／CSS／JavaScript、頁面記憶體假資料、重新整理後重設、角色切換不代表正式授權 |
