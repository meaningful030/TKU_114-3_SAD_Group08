# 更新後的程式碼代理實作任務書

## 0. 任務書使用方式

這是一份可單獨交給程式碼代理的完整指令，不要求代理閱讀聊天紀錄。需求與驗收以本文件列出的編號、規則及固定資料為準；遇到「不可自行決定」事項時，必須保留待確認，不得用常識或生成式 AI 建議補答案。

執行位置：

- 7/13 基線：`../0713_Requirements_Code_Agent/first_working_slice/`
- 本次工作目錄：`revised_or_second_working_slice/`
- 需求與驗收文件位於本任務書同一資料夾。

若工作目錄尚未包含第二切片，從 7/13 基線複製後實作；若工作目錄已包含第二切片，先核對現況，只修正不符合本任務書之處，不得重複建立畫面或改寫已通過的功能。

## 1. 目前系統狀態

### 1.1 7/13 基線狀態

7/13 第一個可操作切片已使用繁體中文原生 HTML、CSS、JavaScript 完成：

1. 攤主填寫攤位名稱、作品類型、email 與選填備註。
2. 必填及基本 email 格式驗證。
3. 成功後顯示申請編號、送件時間與「審核中」。
4. 以角色切換模擬主辦方，讓同一筆申請出現在待審清單。

基線沒有通過／退回、審核意見、攤主結果回查、退回後修改重送及完整可存取語意。

### 1.2 目前工作目錄狀態

目前工作目錄可能已含本次實作結果，檔案應包括：

- `index.html`：申請、狀態與主辦方審核介面。
- `styles.css`：介面樣式。
- `app.js`：畫面事件、角色切換與狀態同步。
- `domain.js`：資料驗證、審核與重送邏輯。
- `domain.test.js`：領域行為檢查，目前基準為 14 項通過。

目前只使用頁面記憶體假資料；重新整理後回到初始狀態。角色切換不是真正登入，不能證明正式身分或權限。

## 2. 本次目標

以 7/13 第一切片為基線，完成或核對第二個可操作切片：

```text
攤主送件

```

本次執行 `IMP-01～IMP-04`，對應 `FR-01～FR-05、FR-08、NFR-04`。`IMP-05／FR-07` 是下一切片，不屬本次程式修改。

## 3. 變更範圍

### 3.1 保留項目

- 保留 FR-01～FR-04 的申請表、驗證、成功結果及待審同步。
- 保留繁體中文介面及「攤主／主辦方」角色切換。
- 保留攤位名稱、作品類型、email、選填備註等既有欄位。
- 作品類型維持**單選**，並保留「其他」及補充類型欄位。
- 保留頁面記憶體假資料與重新整理重設的限制。
- 保留既有通過的 `domain.test.js` 行為，不因重構而退步。

### 3.2 修正項目

- 攤位名稱或退回意見只有空白字元時，一律視為未填。
- 選「其他」但未填補充類型時，拒絕送件並保留其他輸入。
- 退回意見空白或全空白時，拒絕更新申請，狀態與待審計數保持不變。
- 通過時審核意見為選填，不得錯誤要求必填。
- 待審計數只計算狀態為「審核中」的案件，不得計入通過或退回案件。
- 錯誤、送件結果與審核結果更新時，提供可辨識的動態訊息語意。

### 3.3 新增項目

- 主辦方審核區，顯示申請編號、攤名、作品類型、email、備註及審核操作。
- 「通過」及「退回」操作；退回時保存完整審核意見。
- 攤主最新申請狀態區，顯示審核中、通過或退回及審核意見。
- 退回案件的「修改原申請」操作。
- 同一申請編號重送：狀態回到審核中、重送次數加一、待審數增加、目前退回提示清除。
- `domain.js` 與 `domain.test.js`，或等效的可獨立測試領域邏輯；基準需維持 14 項行為檢查通過。
- label、可辨識按鈕名稱及 `aria-live` 或等效動態訊息語意。

### 3.4 刪除項目

- 若存在「通過也必須填審核意見」的規則或驗證，必須刪除。
- 若存在「退回後重送會建立新申請編號」的行為，必須刪除，改為沿用原編號。
- 若工作目錄存在無內容且未被引用的檔案 `1`，刪除該檔；刪除前先確認它確實是空檔且沒有引用。
- 不得刪除備註欄位、既有測試、角色切換或第一切片功能。

### 3.5 本次不做

- `IMP-05／FR-07`：名稱、編號或類型搜尋及靜態地圖，排下一切片。
- `IMP-06／FR-06`：攤位配置與重複格位衝突，排後續切片。
- 正式登入、身分驗證、權限系統、後端、資料庫及持久保存。
- 草稿自動儲存、email／推播通知、金流、候補、多場次、票務、商品管理、R18 審查及工作排班。
- 自動排位、排位最佳化及互動式地圖編輯器。
- 正式效能門檻、指定瀏覽器相容性及伺服器端安全功能。

## 4. 技術與資料限制

| 項目 | 必須遵守的限制 |
|---|---|
| 前端技術 | 原生 HTML、CSS、JavaScript；不得加入框架、套件管理器或外部程式庫 |
| 允許檔案 | `index.html`、`styles.css`、`app.js`、`domain.js`、`domain.test.js`；必要時可調整既有檔案，但不得無理由增加工具鏈 |
| 後端與儲存 | 無後端、無資料庫、無 API、無 `localStorage`／`sessionStorage`；只用目前頁面記憶體 |
| 網路 | 不依賴網路服務；直接開啟 `index.html` 仍可操作 |
| 測試資料 | 只使用虛構資料，例如 APP-001、星屑工房、`owner@example.com`；不得放入真實個資或正式活動資料 |
| 申請狀態 | 只使用「審核中／通過／退回」；不得新增其他正式狀態 |
| 申請編號 | 原型可使用既有編號產生方式；不得宣稱跨工作階段唯一或正式保存 |
| email 政策 | D-06 為所有已登入使用者可查看 email；本期沒有真正登入，因此只可顯示原型資料，不得宣稱已通過權限測試 |
| 可存取性 | 使用 label、原生 button、可辨識名稱及 `aria-live` 或等效語意；不自行擴大為完整 WCAG、手機版或純鍵盤正式驗收 |
| 語言 | 所有使用者介面與錯誤提示使用繁體中文 |

### 6.1 必要驗收判定

- AC-05-02：通過且意見空白時，狀態為通過，待審數 1 → 0。
- AC-05-03：退回且意見為「請補充攤位展示說明」時，完整保存文字，待審數 1 → 0。

- AC-08-03：退回 APP-001 重送後仍為 APP-001，狀態改審核中、重送次數 0 → 1、待審數增加。

## 7. 執行順序

1. **盤點現況**：列出工作目錄檔案並開啟現有畫面；判斷是 7/13 基線或已完成第二切片，不得先整份重寫。
2. **執行 IMP-01**：先跑既有測試與 FR-01～04 回歸；記錄基線失敗，不可掩蓋。
3. **執行 IMP-02**：完成或核對審核資料、通過、退回、空白驗證、狀態與待審計數。
4. **執行 IMP-03**：完成或核對攤主結果回查、修改原申請與同編號重送。
5. **執行 IMP-04**：補足或核對 label、按鈕名稱及動態訊息語意；保持 CON-02 技術限制。
6. **自動驗證**：檢查 JavaScript 語法並執行 `domain.test.js`；預期 14 項行為檢查全部通過。
7. **逐筆驗收**：依本文件第 6 節的 AC 準備資料、操作並記錄通過／失敗／不執行；失敗修正後必須留下重測結果。
8. **範圍檢查**：確認沒有 FR-06、FR-07、登入、儲存、通知、金流、自動排位或互動地圖。
9. **回報**：依第 9 節格式輸出；無法人工執行螢幕閱讀器時列為待人工確認，不得假造通過。

## 8. 完成標準

- IMP-01～IMP-04 的程式修改與靜態檢查完成，或現有實作經核對已符合。
- `domain.test.js` 顯示 14 項行為檢查全部通過；若項目數改變，必須說明新增／刪除原因，不能只回報「測試通過」。
- FR-01～FR-04 回歸無退步。
- 主辦方可通過或退回；通過意見選填，退回空白／全空白被拒絕。
- 攤主可查看最新結果，退回後可修改並以同一申請編號重送。
- 狀態、審核意見、重送次數與待審計數在兩個角色畫面一致。
- 所有現有輸入有 label，動作使用可辨識按鈕，主要動態結果具有 `aria-live` 或等效語意。
- 沒有新增任何本次不做功能，也沒有宣稱正式保存、登入、權限或 NFR 量化目標已完成。
- 自動／靜態驗收有證據；實際螢幕閱讀器人工結果若未執行，明確標示待測。

## 10. 不可自行決定的事項

- 不得決定攤位名稱字數、更多正式申請欄位或複雜 email 格式。
- 不得加入搜尋的部分比對、大小寫處理、作者別名或排序；FR-07 整體屬下一切片。
- 不得設定補件期限、重送上限或建立每次內容版本歷程；目前只保存最新內容及重送次數。
- 不得決定正式審核角色、撤銷審核、審核順序或自動審核判準。
- 不得決定禁排格、多格攤位、特殊衝突、臨時異動或排位優先權。
- 不得把 NFR-02 的 10 筆／100%、NFR-06 的 100 筆／10 次／1 秒、NFR-07 的 Chrome／Edge 當成正式門檻。
- 不得把角色切換當成真正登入；不得自行把 email 政策改回「只有承辦人可看」。D-06 的最新版政策是所有已登入使用者可看，但本期無法驗證。
- 不得把模擬訪談寫成真實使用者訪談，也不得把 AI 建議當成需求來源。
- 不得新增草稿自存、通知、自動排位或任何本次不做功能。
- 若未定義事項會阻擋 IMP-01～04，停止該項並在回報中列為待確認；不得猜答案後繼續。
