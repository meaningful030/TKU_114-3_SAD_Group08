# 程式碼代理實作任務書

## 工具選擇

| 項目 | 內容 |
|---|---|
| 程式碼代理 | OpenAI Codex |
| 使用介面 | 桌面應用程式 |
| 選擇原因 | 可讀取既有章程、建立文件與原型並執行本機檢查 |
| 登入或授權 | 使用者已登入；讀取 Downloads 經使用者授權 |
| 外部服務 | 本次不將真實個資或未公開活動資料放入原型；只使用模擬資料 |
| 資料保護 | 不使用真實姓名、電話、信箱、憑證或正式申請資料 |

## 專案背景與角色

同人誌販售會攤位管理系統，改善人工收件、審核、排位與現場查攤資訊不一致。角色為攤主、主辦方承辦人員與一般參加者；本切片只直接支援前兩者。

## 第一個可操作切片

攤主填寫核心欄位並送出申請；成功後看見申請編號與「審核中」，切換到主辦方檢視時，同一筆申請出現在待審清單。

## 範圍

範圍內：FR-01～FR-04、NFR-01、CON-01；繁體中文介面、角色切換、假資料、表單驗證、成功結果與待審清單。

範圍外：正式登入、後端與資料庫、email／推播、金流、候補、多場次、商品庫存、R18 審查、票務、工作排班、自動排位、互動式地圖、正式個資。

## 需求與驗收條件

| 需求 | 來源 | 驗收條件 | 畫面／操作 |
|---|---|---|---|
| FR-01 | INT-OWN-01、CHA-DATA-01 | 填妥三個核心欄位可送出 | 申請表 |
| FR-02 | INT-OWN-01 | 任一必填缺漏時不得送出並顯示欄位提示 | 送出按鈕／錯誤訊息 |
| FR-03 | INT-OWN-01、CHA-TASK-01 | 成功後顯示唯一申請編號與審核中 | 成功卡片 |
| FR-04 | CHA-TASK-01 | 切換主辦方後，待審清單顯示相同攤名與作品類型 | 待審清單 |
| NFR-01 | FEAS-OPS-01 | 必填標示與結果清楚，不需另讀操作手冊 | 全流程 |

## 技術限制

| 項目 | 設定 |
|---|---|
| 前端 | HTML、CSS、JavaScript |
| 後端 | 無 |
| 儲存 | 僅頁面記憶體假資料，重新整理即重設 |
| 部署 | 不需要，直接開啟 `first_working_slice/index.html` |
| 外部套件 | 不使用 |
| 正式登入 | 不允許，以角色切換模擬 |

## 代理不得自行決定

- 不得新增角色、需求、欄位、審核規則或範圍外功能。
- 不得把模擬訪談描述為真實訪談。
- 不得宣稱資料已正式保存或通知已送出。
- 需求衝突時，以 project_charter_v2.md 與需求來源表為準並標記待確認。
