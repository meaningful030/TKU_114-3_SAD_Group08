# 0713 訪談大綱

## 訪談目的與對象

目的：確認攤位申請欄位、送件回饋、審核與排位規則，以及參加者查攤時真正需要的搜尋資訊。本次為課堂模擬訪談，來源強度標為「中」，後續仍應以真實主辦方、攤主與參加者訪談或正式簡章交叉驗證。

| 受訪角色 | 預計時間 | 想確認的核心任務 |
|---|---:|---|
| 攤主 | 15 分鐘 | 填寫並送出攤位申請、查詢狀態 |
| 主辦方承辦人員 | 20 分鐘 | 審核申請、排位、處理衝突 |
| 一般參加者 | 15 分鐘 | 用名稱、編號或作品類型查攤 |

## 角色一：攤主訪談題組（8 題）

1. 【背景】請描述最近一次申請同人誌販售會攤位的經驗。
   - 追問：當時使用哪些簡章、表單或聯絡管道？
2. 【現況流程】你通常如何完成攤位申請？請按順序說明每一步。
   - 追問：從哪裡取得表單？到哪個結果才算完成送件？
3. 【痛點】申請過程中最花時間或最容易不確定的是哪一步？
   - 追問：最近一次發生時造成什麼影響？
4. 【資料需求】你申請時通常需要準備哪些資料？哪些欄位最難準備？
   - 追問：攤位名稱、作品類型與聯絡方式是否都有明確填寫規則？
5. 【通知或回饋】申請送出後，你目前如何知道是否已收件、正在審核或已完成？
   - 追問：什麼資訊能讓你確認主辦方確實收到申請？
6. 【例外情況】若必填資料缺漏或格式錯誤，你認為合理的處理方式是什麼？
   - 追問：應該阻止送出，還是允許送出後補正？為什麼？
7. 【例外情況／限制】若申請被退回或需要補正，你需要看到哪些原因與後續步驟？
   - 追問：是否存在補正期限或不可修改的資料？
8. 【成功標準】完成送件後看到哪些資訊，會讓你確定這次申請已成功完成？

## 角色二：主辦方承辦人員訪談題組（8 題）

1. 【背景】請描述最近一次處理攤位申請、審核或排位的經驗。
   - 追問：當時使用哪些表單、email、試算表或場地文件？
2. 【現況流程】收到申請後，從收件到完成排位通常依什麼順序處理？
   - 追問：中間需要和哪些人或單位交換資料？
3. 【痛點】哪些步驟最容易漏件、重複處理或需要人工重做？
   - 追問：最近一次錯誤如何發現與補救？
4. 【資料需求】承辦人審核一筆申請時，必須看到哪些欄位或文件？
   - 追問：哪些資料會影響通過、退回或排位決定？
5. 【權限或責任】哪些申請資料只能由承辦人查看？誰可以決定通過、退回與位置調整？
   - 追問：場館方或現場工作人員在流程中負責什麼？
6. 【例外情況／限制】什麼情況算排位衝突？除了同一格重複，是否還有禁排格、多格攤位或相鄰限制？
   - 追問：發現衝突時應保留哪一筆配置，由誰決定如何修改？
7. 【例外情況／回饋】資料不完整、申請退回或位置臨時變更時，目前如何處理及告知相關角色？
   - 追問：哪些情況本期不應由系統自動處理？
8. 【成功標準】若只能先改善一段流程，你會選哪一段？要看到什麼結果才算改善成功？

## 補充角色：一般參加者（選用）

1. 參加者到現場想找特定攤位時，目前怎麼找？
2. 記得的通常是攤位名稱、編號、作者或作品類型中的哪一種？
3. 搜尋不到結果或地圖與現場不一致時，你通常怎麼處理？
4. 查詢結果應包含哪些資訊，才能讓你順利找到攤位？

## 訪談倫理與紀錄

- 訪談開始前說明課堂用途並取得紀錄同意。
- 不記錄真實姓名、聯絡方式或未公開活動資料。
- 模擬內容不得標示為真實訪談；推論需回訪確認。
