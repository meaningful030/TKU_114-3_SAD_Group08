## 圖面判讀

- 系統邊界內共有 UC-01～UC-06，與任務 2 使用案例清單完全一致。
- 系統邊界外共有三種直接參與者：攤主、主辦方承辦人、一般參加者。
- 箭頭由參與者指向使用案例，名稱分別是「提交、查詢、補正、審核、配置、查詢」，表示角色透過系統完成該目標。
- 藍色外框表示 D-02 已確認的最高優先目標；灰色表示既有或目前切片；虛線表示下一或後續切片。
- 圖中沒有資料庫、畫面、欄位、按鈕、內部模組或流程先後。

## 關係選擇

本圖沒有使用 `«include»` 或 `«extend»`：

- 欄位驗證與取得申請編號是 UC-01「提交攤位申請」的必要步驟／成功條件，不另建可被多案重用的獨立使用案例。
- 待審清單是 UC-03「審核攤位申請」的進入方式，不是參與者的獨立終極目標。
- 退回意見空白、重複格位與查無攤位是各案例的例外或替代結果，不另畫 `«extend»`，避免把例外訊息誤當完整使用案例。
- UC-02「查詢申請狀態」與 UC-04「補正退回申請」具有不同目標與成功結果，因此保留為兩個獨立使用案例，而非 include／extend 關係。

若後續需求確認出可由多個完整案例共同呼叫、且本身具明確參與者價值的行為，再評估加入 `«include»`；若出現只在特定條件下附加且具有獨立目標的行為，再評估 `«extend»`。
