## 1. 範圍選擇

### 選擇結果

本次選擇「修正既有切片的一致性問題」，不直接開始 FR-07 第三切片。

### 選擇理由

- MC-01 直接涉及最高優先 FR-05、UC-03 成功後置條件及 AC-05-06。
- 現況程式以 `applications.map(...)` 將所有案件渲染在同一案件表；審核後待審計數雖為 0，通過／退回案件仍殘留。
- 必須同時保留 AC-05-02、AC-05-07 所要求的結果重新開啟，以及 FR-08 的退回補正；不能只把資料列刪除。
- 先穩定「待審／已審／重送」狀態邊界，再加入 FR-07，能避免第三切片建立在錯誤清單語意上。

### 本次成功結果

1. 待審區只顯示「審核中」案件。
2. 通過或退回後，案件離開待審區，待審筆數正確更新為 0。
3. 已審結果仍可由獨立區域重新開啟，以唯讀方式查看狀態與完整意見。
4. 退回案件修改重送後，以同一編號回到待審區；已審區不保留同一案件的重複活動列。
5. 自動測試及人工驗收能重現通過、退回與重送三條路徑。
