## 3. 現況問題與目標改善摘要

| 痛點 | 現況影響 | 目標改善 | 本期狀態 |

## 4. 本次實作選擇與操作方式

### 選擇

採用「模型修正型」，修正 MC-01，不建立 FR-07 第三切片：

- 新增待審及已審的互斥資料投影。
- 待審表只顯示「審核中」。
- 通過／退回移至獨立「已審結果」，可唯讀回查。
- 空白或全空白退回意見不更新狀態。
- 退回案件修改後以同一編號回到待審，不建立副本。

### 操作

1. 開啟 [`revised_or_third_working_slice/index.html`](revised_or_third_working_slice/index.html)。
2. 以「攤主」輸入假資料並送出。
3. 切換「主辦方」，從待審案件開始審核。
4. 保持意見空白可通過；退回時必須輸入至少一個非空白字元。
5. 通過／退回後由已審結果唯讀回查。
6. 退回時切回攤主，修改原申請並重新送出；同一編號回到待審。

資料只存在目前瀏覽器工作階段，重新載入會重設。

## 6. 已知限制與待確認問題

### 已知限制

- 角色切換只供原型示範，不是正式登入或權限驗證。
- 只有記憶體假資料，沒有後端、資料庫或跨工作階段保存。
- `reviewedAt` 只是工作階段示範值，不是正式驗收資料。
- 已審操作停用是原型安全限制，不代表不可撤銷政策已定案。
- 公開搜尋與攤位配置仍是後續切片。

### 待確認

- UQ-01：正式欄位、格式、申請編號與保存方式。
- UQ-02：模糊比對、大小寫、空白處理及作者別名。
- UQ-03：補件期限、重送上限及版本歷程。
- UQ-04／BR-04：正式角色、權限、審核順序、撤銷與重新審核。
- UQ-05：禁排格、多格攤位、特殊衝突及配置權限。
- UQ-06：正式效能、一致率與測試環境門檻。
- `reviewedAt` 的正式格式、時區、保存及驗收方式。
