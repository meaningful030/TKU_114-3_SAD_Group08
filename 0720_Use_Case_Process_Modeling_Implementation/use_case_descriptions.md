- `A-x`：替代流程；每一段均標示從哪個步驟分支及如何結束。

## UC-03 審核攤位申請

### 基本資料

| 欄位 | 內容 |
|---|---|
| 使用案例編號／名稱 | UC-03／審核攤位申請 |
| 主要參與者 | 主辦方承辦人 |
| 支援參與者 | 無；攤主是結果的後續使用者，不在同一次審核操作中互動 |
| 層級與優先 | 使用者目標；Must／最高優先／第二切片完成 |
| 觸發事件 | 主辦方承辦人選取一筆待審申請，準備作成通過或退回決定 |
| 前置條件 | 1. 系統存在一筆狀態為「審核中」的申請。2. 固定驗收案件為 APP-001，待審筆數為 1。3. 原型目前以主辦方角色示範；正式審核權限仍屬 UQ-04。 |
| 成功後置條件 | APP-001 狀態更新為「通過」或「退回」；依 BR-02 保存允許的審核意見；案件離開待審清單，待審筆數 1→0；結果與意見可重新開啟回查。 |
| 最低保證 | 若退回意見無效，申請仍為「審核中」，原審核結果與意見不變，待審筆數仍為 1，且顯示可理解的錯誤；不得產生負數或遺失案件。 |
| 對應需求 | US-05、US-06；FR-04、FR-05；DR-02 |
| 適用業務規則 | BR-02：退回意見必填，通過意見選填。BR-04 仍為待確認，不放入已確認流程。 |
| 對應驗收條件 | AC-05-01～AC-05-07；另有 AC-NFR-04-01 的螢幕閱讀器人工驗收待執行 |

### 輸入、輸出與狀態變化

| 類型 | 內容 |
|---|---|
| 輸入 | 被選取的申請編號、審核動作（通過／退回）、選填或必填的審核意見 |
| 固定測試輸入 | APP-001；星屑工房；原創漫畫；`owner@example.com`；需要靠牆位置；有效退回意見「請補充攤位展示說明」 |
| 輸出 | 申請核心資料、可執行審核動作、成功結果或錯誤提示、更新後狀態、審核意見及待審筆數 |
| 成功狀態變化 | `審核中 → 通過` 或 `審核中 → 退回`；待審筆數 `1 → 0` |
| 例外狀態變化 | 無；狀態保持「審核中」，待審筆數保持 1，原資料不變 |

### 主要流程 M：通過申請且意見留空

| 步驟 | 執行者 | 行為與可觀察結果 |
|---|---|---|
| M-1 | 主辦方承辦人 | 進入待審工作入口。 |
| M-2 | 系統 | 顯示待審筆數 1，清單中有 APP-001，狀態為「審核中」。 |
| M-3 | 主辦方承辦人 | 選取 APP-001 開始審核。 |
| M-4 | 系統 | 顯示 APP-001、星屑工房、原創漫畫、`owner@example.com`、需要靠牆位置，以及「通過」「退回」操作。 |
| M-5 | 主辦方承辦人 | 保持審核意見空白，選擇「通過」。【分支點：若選擇退回，走 A-1；若通過時填有意見，走 A-2】 |
| M-6 | 系統 | 依 BR-02 接受空白通過意見，將 APP-001 更新為「通過」。 |
| M-7 | 系統 | 保存通過結果，待審筆數由 1 更新為 0，待審清單不再顯示 APP-001，且不得顯示負數。 |
| M-8 | 主辦方承辦人 | 重新開啟 APP-001 的結果。 |
| M-9 | 系統 | 顯示 APP-001 為「通過」，審核意見為空。 |

**結束方式：成功。**符合 AC-05-01、AC-05-02、AC-05-06；成功後置條件成立。

### 替代流程

#### A-1：附有效意見退回

- **分支點：M-5。**承辦人不選通過，改輸入「請補充攤位展示說明」並選擇「退回」。
- A-1.1 系統確認意見至少包含一個非空白字元，不是空字串或全空白。
- A-1.2 系統將 APP-001 更新為「退回」，逐字保存「請補充攤位展示說明」。
- A-1.3 系統將待審筆數由 1 更新為 0，案件離開待審清單。
- A-1.4 承辦人重新開啟結果；系統顯示「退回」及逐字相同的審核意見。

**結束方式：替代成功。**符合 AC-05-03、AC-05-07；成功後置條件以「退回」結果成立。

#### A-2：通過並填寫選填意見

- **分支點：M-5。**承辦人填寫非空白意見後選擇「通過」。
- A-2.1 系統依 BR-02 接受通過意見為選填資料。
- A-2.2 系統保存「通過」結果及已輸入意見，將待審筆數由 1 更新為 0。

**結束方式：替代成功。**回到 M-8 驗證結果可回查；FR-05 與 BR-02 的成功後置條件成立。

### 例外流程

- E-1.1 系統拒絕更新，顯示「退回時請填寫審核意見」或同義的明確提示。
- E-1.2 系統保持 APP-001 為「審核中」，待審筆數保持 1，原意見與結果不變。

**結束方式：例外已處理、使用案例未成功；可由 M-5 重新輸入。**符合 AC-05-04 與最低保證。

#### E-2：退回意見只有空白字元

- **分支點：A-1 的第一步／M-5。**承辦人輸入 3 個空白字元後選擇「退回」。
- E-2.1 系統將全空白內容視為未填，拒絕更新並顯示必填提示。
- E-2.2 系統保持 APP-001 為「審核中」，待審筆數保持 1，原意見與結果不變。

**結束方式：例外已處理、使用案例未成功；可由 M-5 重新輸入。**符合 AC-05-05 與最低保證。

## UC-04 補正退回申請

### 基本資料

| 欄位 | 內容 |
|---|---|
| 使用案例編號／名稱 | UC-04／補正退回申請 |
| 主要參與者 | 攤主 |
| 支援參與者 | 無；主辦方承辦人會在本案例成功後，於後續 UC-03 處理重送案件 |
| 層級與優先 | 使用者目標；Must／第二切片完成；使用者選為第三份完整描述 |
| 觸發事件 | 攤主看到申請為「退回」及審核意見後，選擇修改原申請並重新送出 |
| 前置條件 | 1. 同一工作階段存在原申請 APP-001。2. APP-001 狀態為「退回」。3. 審核意見為「請補充攤位展示說明」。4. 固定驗收資料的重送次數為 0。 |

| 對應需求 | US-04；FR-08；DR-03 |
| 適用業務規則 | BR-03：只有退回申請可修改原申請重送；重送沿用編號、回到審核中且重送次數加一。 |
| 對應驗收條件 | AC-08-01～AC-08-03；前置退回結果另對應 AC-05-03、AC-05-07 |

| 類型 | 內容 |
|---|---|
| 輸入 | 原申請 APP-001、退回審核意見、攤主修改後的最新申請內容、重新送出動作 |
| 輸出 | 退回狀態及完整意見、載入的原申請資料、重送成功結果、原申請編號、審核中狀態、重送次數及待審結果 |
| 成功狀態變化 | APP-001：`退回 → 審核中`；重送次數 `0 → 1`；待審筆數 `+1`；目前退回提示清除 |
