# 任務 12：第二切片一致性修正版

本切片修正 MC-01，將主辦方的「待審案件」與「已審結果」分流。

## 開啟方式

直接以現代瀏覽器開啟 `index.html`。資料只存在目前工作階段，重新載入會清除。

## 可操作流程

1. 以「攤主」送出假申請。
2. 切換「主辦方」，從待審案件開始審核。
3. 選擇通過，或輸入意見後退回。
4. 通過／退回案件會離開待審區，移至已審結果並可唯讀回查。
5. 退回時切回攤主，可修改原申請並以同一編號重送。

## 測試

```powershell
node .\domain.test.js
```

`ui.acceptance.test.js` 需要 Node.js、Playwright 及 Chromium 相容瀏覽器；它不影響 `index.html` 單獨運作。

## 範圍限制

不含正式登入、付款、通知、後端、資料庫、攤位配置或地圖搜尋。`reviewedAt` 只作工作階段示範，不是正式驗收資料。
