# 任務 8：流程資料交換表

需求基準：`0.8-draft／2026-07-15`  
流程基準：任務 6「系統輔助收件與查攤，保留人工審核及排位」  
用途：作為下一單元系統情境圖與第 0 層資料流程圖（DFD）的輸入。

## 1. 建模原則

- 表中的「資料去向」使用概念資料集合，例如申請資料、待審資料、審核結果及配置資料；不表示目前已建立資料庫。
- 依 `CON-01`，第二個可操作切片只有頁面記憶體假資料，重新整理後會重設；不得從本表推論已有後端、正式登入或持久保存。
- 申請狀態統一使用「審核中／通過／退回」。配置資料的新增或修改另列為配置狀態，不自行增加「已配置」申請狀態。
- 未確認的正式欄位、位置格式、保存方式與歷程均保留 UQ／DR 註記，不當成已定案資料。
