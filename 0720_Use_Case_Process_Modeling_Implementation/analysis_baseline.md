### 2.2 本期品質與限制

- NFR-04「螢幕閱讀器基本支援」是已確認的本期 Must；靜態語意檢查完成，但實際螢幕閱讀器人工驗收仍待執行。
- NFR-05／CON-02 的原生 HTML、CSS、JavaScript、無外部套件及可離線開啟是既有技術基準。
- NFR-03 本期只可確認「公開地圖／公開配置不得含 email」；正式登入後的權限行為不在本期驗收範圍。
- NFR-01 可沿用目前清楚顯示錯誤與成功結果的最小行為；正式欄位與格式規則仍待確認。
- NFR-02 隨 FR-07 分析，但「至少 10 筆、100%」仍是候選目標，不得畫成已確認品質門檻。
- NFR-06、NFR-07、NFR-08 保留為候選或後續評估，不得因已有草稿文字就視為本期正式完成標準。

正式登入與身分驗證、後端、資料庫、持久保存、通知、金流、候補、多場次、票務、商品管理、R18 審查、工作排班、自動排位、排位最佳化及互動式地圖均不納入。角色切換只代表原型展示視角，不代表正式授權。

## 6. 全模型統一術語與畫法

| 類別 | 統一術語 |
|---|---|
| 系統 | 同人誌販售會攤位管理系統「攤集」 |
| 使用者角色 | 攤主、主辦方承辦人、一般參加者 |
| 支援利害關係人 | 現場工作人員；不是本期直接操作角色 |
| 申請狀態 | 審核中、通過、退回 |
| 主要物件 | 攤位申請、審核資料、攤位配置、靜態地圖／搜尋索引 |
| 地圖 | 靜態平面圖＋格子編號＋查詢結果標記 |
| 排位 | 主辦方手動配置；系統只做已確認的衝突檢查，不做自動排位 |
| 身分表示 | 角色切換／原型展示視角；不得稱為登入或權限驗證 |
| 不確定標記 | `【待確認：UQ-xx】`；可放在註解、替代／延伸片段或模型外的問題清單，不得當成正常主流程 |

模型完成前應核對：標題或註記含 `需求包 0.8-draft／2026-07-15`、角色與狀態名稱完全一致、待確認分支有 UQ 編號、沒有把角色切換畫成登入，也沒有把 FR-06／FR-07 的規格存在誤畫成目前切片已完成。
